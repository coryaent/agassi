rqlite is a lightweight, distributed relational database, which uses SQLite as its
storage engine. It provides an easy-to-use, fault-tolerant store for relational data.

Usage: rqlited [arguments] <data directory>
  -auth string
        Path to authentication and authorization file. If not set, not enabled
  -cpu-profile string
        Path to file for CPU profiling information
  -disco-id string
        Set Discovery ID. If not set, Discovery Service not used
  -disco-url string
        Set Discovery Service URL (default "http://discovery.rqlite.com")
  -dsn string
        SQLite DSN parameters. E.g. "cache=shared&mode=memory"
  -expvar
        Serve expvar data on HTTP server (default true)
  -http-addr string
        HTTP server bind address. For HTTPS, set X.509 cert and key (default "localhost:4001")
  -http-adv-addr string
        Advertised HTTP address. If not set, same as HTTP server
  -http-ca-cert string
        Path to root X.509 certificate for HTTP endpoint
  -http-cert string
        Path to X.509 certificate for HTTP endpoint
  -http-key string
        Path to X.509 private key for HTTP endpoint
  -http-no-verify
        Skip verification of remote HTTPS cert when joining cluster
  -join string
        Comma-delimited list of nodes, through which a cluster can be joined (proto://host:port)
  -join-attempts int
        Number of join attempts to make (default 5)
  -join-interval string
        Period between join attempts (default "5s")
  -mem-profile string
        Path to file for memory profiling information
  -node-ca-cert string
        Path to root X.509 certificate for node-to-node encryption
  -node-cert string
        Path to X.509 certificate for node-to-node encryption (default "cert.pem")
  -node-encrypt
        Enable node-to-node encryption
  -node-id string
        Unique name for node. If not set, set to hostname
  -node-key string
        Path to X.509 private key for node-to-node encryption (default "key.pem")
  -node-no-verify
        Skip verification of a remote node cert
  -on-disk
        Use an on-disk SQLite database
  -pprof
        Serve pprof data on HTTP server (default true)
  -raft-addr string
        Raft communication bind address (default "localhost:4002")
  -raft-adv-addr string
        Advertised Raft communication address. If not set, same as Raft bind
  -raft-apply-timeout string
        Raft apply timeout (default "10s")
  -raft-election-timeout string
        Raft election timeout (default "1s")
  -raft-log-level string
        Minimum log level for Raft module (default "INFO")
  -raft-non-voter
        Configure as non-voting node
  -raft-open-timeout string
        Time for initial Raft logs to be applied. Use 0s duration to skip wait (default "120s")
  -raft-remove-shutdown
        Shutdown Raft if node removed
  -raft-snap uint
        Number of outstanding log entries that trigger snapshot (default 8192)
  -raft-snap-int string
        Snapshot threshold check interval (default "30s")
  -raft-timeout string
        Raft heartbeat timeout (default "1s")
  -version
        Show version information and exit
